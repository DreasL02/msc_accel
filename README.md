# msc_accel

This folder contains the source tree for the UVM co-emulation / acceleration setup used in the thesis 'Accelerating UVM testbenches using co-emulation in FPGAs'.

The repository is split into a few self-contained areas to be deployed across the development environment. 

See Figure below for a high-level overview of the components in the development environment:

![High-level overview of the development environment](Diagrams/devenv.svg)


## Top-level directory layout

- [msc_uvm_accelerator/](msc_uvm_accelerator/) - UVM package and DPI bridge of the host side of the co-emulation. Built and expanded on elements of the [Amiq OFC UVM co-emulation framework](https://github.com/amiq-consulting/amiq_ofc). Host machine.

- [msc_vitis/](msc_vitis/) - Xilinx Vitis workspace content for the Zynq side, including the platform and example applications. Development machine.


- [msc_testbench/](msc_testbench/) - Board-level co-emulation RTL infrastructure, testbench for evaluating framework localhost, and simulator-facing glue. Development machine.

- [msc_apb_example/](msc_apb_example/) - APB example designs and testbenches used as case studies for the accelerator flow. Host machine/Development machine.

- [msc_zynq_programming/](msc_zynq_programming/) - Scripts and ELF files used to program the Zynq target. Also present is the xsa file used for the Vitis project. Deployment machine.

- [msc_vivado/](msc_vivado/) - Contains Tcl file for building Vivado block design for PS-GEM0 option with ILA.

## Links to most interesting files and folders

- [msc_uvm_accelerator/sv/](msc_uvm_accelerator/sv/) - SystemVerilog/UVM side of the host side of the co-emulation interface.

- [msc_uvm_accelerator/cpp/client.cpp](msc_uvm_accelerator/cpp/client.cpp) - C++ side of the DPI-C bridge that implements the network protocol and handles communication with the Zynq target.

- [msc_vitis/apb_tcp/src/freertos_tcp_perf_server.c](msc_vitis/apb_tcp/src/freertos_tcp_perf_server.c) - FreeRTOS TCP server that runs on the Zynq target and handles requests and responses from the host side of the co-emulation interface using the DMA and various queues. Built on the example FreeRTOS TCP server provided by Xilinx in the Vitis installation. This can be found in [msc_vitis/echo_server/src/freertos_tcp_perf_server.c](msc_vitis/echo_server/src/freertos_tcp_perf_server.c).

- [msc_vitis/apb_tcp/src/accel_dma.c](msc_vitis/apb_tcp/src/accel_dma.c) - Custom Xilinx AXI DMA management code.

- [msc_testbench/sv/](msc_testbench/rtl/) - Custom RTL components for the co-emulation setup. In subdirectory [/general_infrastructure](msc_testbench/sv/rtl/general_infrastructure) there is the common infrastructure. The remaining files are the APB case study RTL components.

- [msc_testbench/hdl/](msc_testbench/hdl/) - Testbench for testing the custom RTL components and framework in simulation.

- [msc_testbench/sim/cpp/board_model.cpp](msc_testbench/sim/cpp/board_model.cpp) - C++ side used by the testbench to model the Zynq in simulation.

- [msc_apb_example/rtl/](msc_apb_example/rtl/) - APB example designs used as case studies. [apb_register_file](msc_apb_example/rtl/apb_register_file.sv) corresponds to Testbench A and B designs, [example_design](msc_apb_example/rtl/example_design.sv) is the Testbench D design, and [fat_design](msc_apb_example/rtl/fat_design.sv) is the Testbench C design using the automatically generated Neural Network located in [AutomaticGeneration](msc_apb_example/rtl/AutomaticGeneration.sv).

- [msc_apb_example/reg_file/](msc_apb_example/reg_file/) - Contains the UVM testbench for Testbench A, B and C. Note the APB UVC is not provided.

- [msc_apb_example/pass_through/](msc_apb_example/pass_through/) - Contains the UVM testbench for Testbench D. Note the APB UVC is not provided.

- [msc_zynq_programming/ali_program.tcl](msc_zynq_programming/ali_program.tcl) - Vivado TCL script to program the Zynq target with the ELF files generated by Vitis.
